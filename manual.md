# 🎙️ 音声文字起こしツール 使い方マニュアル

> このマニュアルは、ITの知識がない方でも使えるように書かれています。  
> **ダウンロードして解凍するだけで、すぐに使い始めることができます！**
> 順番通りに進めれば必ず使えるようになりますので、安心して読み進めてください。

---

## 📋 このツールでできること

PCから流れている音（会議・講義・動画など）を録音して、  
**自動で文字に起こし、PDFファイルとして保存**してくれるツールです。

- つなぎ言葉（「えー」「あのー」等）は**自動的に除去**されます。
- AIが内容から**自動でタイトル**を付けてくれます。
- ファイルサイズが大きくても自動で圧縮して処理します。

```
PC音声 → 自動録音 → Geminiが文字起こし → PDFファイル完成！
```

## 1. 最初だけやること（初期設定）

> [!IMPORTANT]
> この手順は**最初の1回だけ**でOKです。2回目以降は不要です。所要時間は約5分です。

### Gemini APIキーを取得する

APIキーとは、AI（Gemini）を利用するための自分専用の「鍵（パスワード）」です。  
> [!NOTE]
> クレジットカードの登録などは不要で、通常利用の範囲なら**完全無料**で使えます！

1. ブラウザで以下のリンクを開きます：  
   👉 **https://aistudio.google.com/apikey**
2. ご自身のGoogleアカウントでログインします。
   - ⚠️ **重要**：会社や学校から支給されたアカウント（@〇〇.co.jp など）だと、管理者の制限でキーが作れない場合が非常に多いです。エラーになった場合は、画面右上のアイコンから**個人のGmailアカウント（@gmail.com）に切り替えて**から再度お試しください。
3. 画面内にある青いボタン **「Create API key (APIキーを作成)」** をクリックします。
4. 選択肢が出た場合は、「Create API key in a new project」をクリックします。
5. 画面に `AIzaSy...` から始まる長い文字列が表示されます。これがあなたのAPIキーです。
   - 横にある **「Copy」** ボタンを押してコピーします。

これで初期設定（準備）は完了です！
このAPIキーは、アプリを初めて起動したときに画面に入力するので、そのままコピーしておいてください。

## 2. 黒い画面（CUI）から使う（リアルタイム録音）

コマンドライン（黒い画面）上でキーボード操作して録音・文字起こしを行う方法です。

### 起動方法

**`run_transcriber.bat`** をダブルクリックします。

### 録音の操作

画面に以下のメッセージが表示されます：

```
>>> Enterキーを押して録音を開始...
```

1. **Enterキーを押す** → 🔴 録音開始！
2. PCから音を出す（動画を再生する・会議に参加するなど）
3. 録音を終わらせたいときに **もう一度Enterキーを押す**

### 自動処理を待つ

Enterを押すと、あとは自動で以下が進みます：

```
録音停止中...
録音保存: output\recording_20260623_103000.wav
  (録音: 48000Hz 2ch → 保存: 16000Hz モノラル)
Gemini APIに接続中... (モデル: gemini-3.5-flash)
音声ファイルをアップロード中...
Geminiで文字起こし中...
文字起こし完了！
PDFを生成中...
PDF保存完了！
```

完了したら、**`output` フォルダ**の中にPDFが保存されています。

## 3. すでにある音声ファイルを文字起こしする

録音済みのファイル（mp3・wav・m4a・mp4など）を文字起こしする方法です。

### 使い方：ドラッグ＆ドロップ（一番簡単）
音声ファイルや動画ファイルを **`run_transcriber.bat`** のアイコンの上に**ドラッグ＆ドロップ**するだけ！自動で文字起こし＆PDF保存が始まります。

---

## 4. 🌟 動画からの「フレーム解析（スライド抽出）」を使う

動画ファイルやパソコンの画面録画から、**「重要なシーン（スライド画面）」をAIが自動抽出し、文字起こしと一緒に要約する**超強力な機能です。

### パターンA：手元の動画ファイルから抽出する
動画ファイル（`.mp4`, `.webm` など）を **`run_transcriber.bat`** に**ドラッグ＆ドロップ**します。
黒い画面が表示され、以下のように聞かれます。
`キースライド（スライド画像）も抽出しますか？ [Y:する / N:しない（音声のみ）]`
キーボードの **Y** を押すと、動画からのスライド抽出と音声の文字起こしが全自動で行われます！

### パターンB：画面を録画しながら抽出する（Web会議などに最適）
**`run_screen_recorder.bat`** をダブルクリックします。
黒い画面が開き、あなたのパソコン画面の録画とPC音声の録音が同時にスタートします。
終了時に再度 Enter を押すと、録画した動画の中から重要なスライド画面だけをAIが自動で探し出し、文字起こしと連動した「リッチ議事録」を作成します。

> [!NOTE]
> スライド抽出機能を利用すると、通常のPDFの代わりに、スライドの要約テキストが追加された **PDF議事録** (`[AI生成タイトル]_日時.pdf`) と、マークダウン形式の議事録（`[AI生成タイトル]_日時.md`）が出力されます。

## 5. 出力されたファイルの見方

処理が完了すると、`output` フォルダに以下のファイルが保存されます：

| ファイル名 | 内容 |
|---|---|
| `recording_日時.wav` | 録音した音声ファイル（録音モードのみ） |
| `タイトル_日時.pdf` | 📄 **文字起こし結果のPDF（メインの成果物）** |

### PDFの内容

- **ヘッダー**：作成日時、音声ファイル名、使用したモデル名
- **本文**：文字起こしの全文テキスト（つなぎ言葉は自動除去済み）

> [!NOTE]
> タイトルはAIが文字起こし内容から自動生成します。  
> PDFファイル名もそのタイトルが使われます（例：`第3回チーム会議_20260623_103000.pdf`）。

## 6. うまくいかないときのチェックリスト

### ❌「APIキーが設定されていません」と表示される
- [ ] アプリ起動時にAPIキーを正しく入力（保存）しましたか？
- [ ] 入力したキーに余計なスペースが含まれていませんか？

### ❌「ファイルが見つかりません」と表示される
- [ ] ファイルのパスに**日本語のスペースや全角文字**が含まれていませんか？
- [ ] ファイルが本当にその場所にありますか？（エクスプローラーで確認）
- [ ] パスを `"` （ダブルクォーテーション）で囲みましたか？

### ❌「ループバックデバイスが見つかりません」と表示される
- [ ] PCのスピーカーやヘッドフォンが接続されていますか？
- [ ] WindowsのサウンドがBluetoothヘッドセット等に切り替わっていませんか？
- Windowsの「サウンドの設定」→「出力デバイス」が通常のスピーカーになっているか確認してください。

### ❌ 文字起こしの精度が悪い・意味不明なテキストが出る
- [ ] 音声の音量は十分でしたか？（小さすぎると認識率が下がります）
- [ ] 無音の録音をしていませんか？（PCから音が出ていない状態で録音するとAIが誤った文字を生成します）

### ❌ 文字起こし結果が空になる
- 使用しているモデルが混雑している可能性があります。しばらく待ってから再試行してください。
- 音声ファイルの長さが極端に短い（数秒以下）場合、正しく処理できないことがあります。

## 👥 他の人にこのツールを配る方法

このツールは**フォルダごと他の人に渡す**（USBメモリやZIP圧縮ファイルなど）だけで共有できます。

- **設定変更は不要**：バッチファイルが自分の場所を自動で判別するため、フォルダをPCのどこに置いてもそのまま動きます。
- **注意点**：
  - 渡す前に `output` フォルダは削除するか中身を空にしておきましょう（個人情報保護のため）。
  - **APIキーは各自で取得・設定する**必要があります（共有しないでください）。
  - 受け取った人は初期設定（APIキーの取得と設定）を行う必要があります。
